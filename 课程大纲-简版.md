# 课程大纲：掀起 AI 编程的"引擎盖" — 从机制拆解到模型复现

**基础模型**：Qwen2.5-Coder-7B-Instruct（7.61B，Dense，Apache 2.0）
**硬件需求**：单卡 A800（80GB）
**复现目标**：在 7B 级别编程代理中达到 SOTA
**复现路径**：Prompt Engineering → LoRA Fine-tune → Agent 架构

---

## 第一篇：机制拆解（模块 0-2）

### 模块 0：核心循环 — AI 编程代理的"发动机"
- 0.1 从真实任务说起：Trace 回放 Claude Code 修 bug 的完整过程
- 0.2 核心循环：Intent → Grounding → Planning → Action → Grounding → Feedback
- 0.3 MDP 建模：状态=代码库快照，动作=工具调用，奖励=测试结果
- 0.4 **Claude Code ↔ Opus 4.6 交互协议**（10 层）
- 0.5 Opus 4.6 能力清单：哪些是模型能力，哪些是工程设计

**交互协议 10 层分类**：

| 协议层 | 核心内容 |
|--------|---------|
| 1. System Prompt | XML 结构、指令重复、CLAUDE.md 加载 |
| 2. Tool Definitions | 15+ 工具的 JSON Schema（Read/Write/Edit/Bash/Grep/Glob/Agent/MCP...） |
| 3. Tool Use | tool_use → tool_result 循环，并行调用 |
| 4. Message 结构 | user/assistant/system 三角色，tool_result 必须在首位 |
| 5. Streaming | SSE 协议：message_start → content_block_delta → message_stop |
| 6. Context Management | 200K 窗口，60% 降级，Auto-Compact + Subagent 隔离 |
| 7. Permission | 5 种模式（default/plan/acceptEdits/dontAsk/bypass），工具级限制 |
| 8. Hooks | 18 种事件（PreToolUse/PostToolUse/UserPromptSubmit...），4 种 handler |
| 9. MCP 集成 | `mcp__<server>__<action>` 命名，settings.json 配置 |
| 10. Subagent | 独立 transcript，主代理只收摘要，可 worktree 隔离 |

**数据流**：
```
User Input → [UserPromptSubmit Hook] → Build Message + System Prompt
→ Claude API (streaming) → Parse tool_use → [PreToolUse Hook]
→ Permission Check → Execute Tool → [PostToolUse Hook]
→ Stream tool_result → Model Response → [Stop Hook] → User
```

## 课程形式

- **4 大篇章，12 个模块**（模块 0-11）
- 每模块 2.5-3 小时（共约 28-33 小时）
- 每模块结构：
  - 机制讲解（25%）：这个环节是什么、为什么重要
  - Opus 4.6 能力分析（20%）：它是怎么做到的
  - Qwen2.5-Coder-7B 现状（15%）：差距在哪
  - 复现路径（20%）：怎么补齐差距
  - 动手实操（20%）：实际编码和实验
- **结业项目**：从零复现一个在 7B 级别达到 SOTA 的编程代理

---

## 课程特色

1. **双线并进**：机制拆解 + 复现实战，理论与实践紧密结合
2. **以终为始**：从 Opus 4.6 的能力反推实现路径，目标明确
3. **可落地**：基于 Qwen2.5-Coder-7B + 单卡 A800，成本极低，学员可独立复现
4. **完整闭环**：从 prompt 到 fine-tune 到 agent 架构，覆盖全流程
5. **前沿视角**：不只是复现现有能力，还探讨未来方向（MCTS、多模态、超长上下文）
